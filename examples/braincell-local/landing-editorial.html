How it works
Three phases. Zero effort.

A VS Code lifecycle hook loads memory before your first message and saves it after each session.

1

Session starts

The VS Code SessionStart hook runs braincell-session-start.ps1. It reads all braincell-lite-*.json files and injects them as a system message before your first chat.

2

You work as normal

Copilot is aware of your project's history. When you make a decision or fix a bug, ask Copilot to save it. Entries are structured, searchable JSON in .braincell/.

3

Next session picks up

Everything saved — decisions, tasks, errors, patterns — is available immediately in the next session. No copy-pasting, no re-explaining. Just open a chat and continue.

Install
One command to get started

Run from your repo root. The installer creates all required files and optionally commits them.

Option A — directly from GitHub (recommended) 
# Run from your repo root in PowerShell
irm https://raw.githubusercontent.com/ITlusions/ITL.BrainCell/main/examples/braincell-local/install-braincell-lite.ps1 | iex
Option B — local file & flags 
.\install-braincell-lite.ps1
.\install-braincell-lite.ps1 -SkipCommit
.\install-braincell-lite.ps1 -TargetPath C:\repos\project
.\install-braincell-lite.ps1 -Force
Files created by installer

  │   └── braincell-lite-notes.json ← initial memory store
  └── .github/

Memory topics
Scoped, extensible, readable

Each topic is a separate JSON file. New topics are created automatically when first used.

braincell-lite-notes.json

Notes

Context, summaries, diagrams

braincell-lite-decisions.json

Decisions

Architecture and design choices

braincell-lite-tasks.json

Tasks

Open, done, and blocked items

braincell-lite-errors.json

Errors

Bugs found and fixed

braincell-lite-patterns.json

Patterns

Reusable commands and solutions

braincell-lite-facts.json

Facts

Versions, URLs, metadata

braincell-lite-[custom].json

Custom topics

Add any domain-specific topic — auth, api, infra, security, deployments. New files are created on first use.

Your data.
Your machine.
Full stop.

BrainCell Lite has no network component. No telemetry. No account required. Memory lives as plain text files in your repository.

Fully local storage

All memory files are plain JSON in your repo. Nothing is sent anywhere.

You control what is saved

Copilot only saves entries when you ask. Sensitive files can be excluded via .gitignore.

Zero telemetry

No analytics, no backend, no external API. BrainCell Lite is a static set of files.

Open source, auditable

MIT licensed. Read every line. No hidden logic, no obfuscation.
