Features
Everything you need, nothing you don't

BrainCell Lite is intentionally minimal — but powerful where it counts.

SessionStart hook — memory loads automatically

A VS Code lifecycle hook fires before your first message and injects all braincell-lite-*.json files as context. You never need to manually reference memory files — it just works.

Structured memory topics

Each topic lives in its own JSON file — tasks, decisions, errors, patterns, facts. Easy to read, diff, and search.

Zero dependencies

Pure PowerShell + plain JSON. No npm, no pip, no container — works anywhere Git runs.

SKILL.md — self-updating

The memory skill downloads the latest version from GitHub weekly, keeping your procedures up to date automatically.

Privacy by default

Nothing leaves your machine. No telemetry, no analytics, no external API. Local JSON, version controlled with your code.

Team-friendly

Commit .braincell/ to share memory across machines and team members. Perfect for onboarding.

How it works
Three phases, zero effort

BrainCell Lite hooks into VS Code lifecycle events to load and save memory transparently.

Session start

Memory loads automatically

The SessionStart hook reads all braincell-lite-*.json files and injects them as a system message before you type anything.

During session

Copilot writes to memory

Decisions, tasks, errors, and patterns are saved as entries in topic-scoped JSON files. A fast-search index is maintained automatically.

Next session

Context carries over

Architecture decisions, open tasks, working commands, and preferences are available immediately — no re-explaining needed.

braincell-session-start.ps1 
# SessionStart hook — fires before first message

$ $files = Get-ChildItem .braincell -Filter "*.json"

✓ braincell-lite-patterns.json   (7 saved)

# Injecting as system context...

✓ Fast-search index: 36 entries

Install
One command, 30 seconds

Run from your repo root. The installer creates all required files and commits them.

1

Run the installer

Open a terminal in your repository root and run the command on the right.

2

Confirm the commit

The installer offers to commit all generated files to your repository.

3

Open a new Copilot Chat

Memory is active immediately. No additional configuration required.
