How it works
Three phases, zero effort

BrainCell Lite hooks into VS Code lifecycle events to load and save memory automatically.

Session start

Memory loads automatically

A VS Code SessionStart hook reads all braincell-lite-*.json files and injects them as context before your first message.

→
During session

Copilot writes to memory

Decisions, tasks, errors, and patterns are saved to topic-scoped files in .braincell/ as you work. An index is maintained automatically.

→
Next session

Continues where you left off

Architecture decisions, open tasks, working commands, and preferences are available instantly — no re-explaining needed.

Install
One command to get started

Run the installer from your repo root. It creates all required files and optionally commits them.

1

Run the installer

Open a terminal in your repository root and paste the command on the right. Use -TargetPath to install into a different directory, -SkipCommit to skip git, or -Force to overwrite existing files.

2

Answer the commit prompt

The installer creates all required files and offers to commit them to your repository.

3

Open a new Copilot Chat

Memory is active immediately — no configuration required. The SessionStart hook loads all memory files automatically.

Requirements
Minimal dependencies by design
□

VS Code + GitHub Copilot

Any Copilot plan. VS Code v1.90+ recommended for SessionStart hook support.

✓

PowerShell 5.1 or later

Pre-installed on Windows. Install pwsh for macOS/Linux via the official package.

●

A local Git repository

Memory files are committed and version-controlled alongside your code.

∞

No other dependencies

No npm packages, no Python, no containers. Pure PowerShell + JSON.

Memory topics
Scoped, extensible, readable

Each topic is a separate JSON file created automatically when needed. Add custom topics for your domain.

braincell-lite-notes.json

Notes

Context, summaries, diagrams

braincell-lite-decisions.json

Decisions

Architecture and design choices

braincell-lite-tasks.json

Tasks

Open, done, and blocked work items

braincell-lite-errors.json

Errors

Bugs found and fixed with solutions

braincell-lite-patterns.json

Patterns

Reusable commands and code solutions

braincell-lite-facts.json

Facts

Versions, URLs, project metadata

braincell-lite-preferences.json

Preferences

Style rules and team conventions

braincell-lite-[custom].json

Custom

Any domain: auth, api, infra, security…

Privacy
Your data stays in your repo
🔒

Fully local storage

All memory files are plain JSON in your repo. Nothing is sent to any external service or server.

👁

You control what is saved

Copilot only saves entries when instructed. Sensitive files can be excluded via .gitignore.

∅

Zero telemetry

No analytics, no backend, no account required. BrainCell Lite has no network component at runtime.
