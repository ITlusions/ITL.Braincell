How it works

Session start

Context loads automatically

A VS Code lifecycle hook runs at startup and injects all your memory files as context into the Copilot Chat session — before your first message.

During session

Copilot writes to memory

Decisions, tasks, errors, and patterns are saved to topic-scoped JSON files in .braincell/ as you work. A fast-search index is maintained automatically.

Next session

Continues where you left off

Architecture decisions, open tasks, working commands, and project preferences are instantly available in the next conversation — no re-explaining needed.

Requirements

+
Visual Studio Code Any recent version with GitHub Copilot Chat extension
+
GitHub Copilot Reads copilot-instructions.md and VS Code lifecycle hooks automatically
+
A Git repository Memory files are version-controlled with your project
+
PowerShell 5.1+ or PowerShell 7+ Built into Windows; install pwsh on macOS/Linux for the SessionStart hook

Install

1

Run the installer from your repo root

Open a terminal in your repository root and run one of the following commands.

Option A — directly from GitHub (recommended):

powershell 
irm https://raw.githubusercontent.com/ITlusions/ITL.BrainCell/main/examples/braincell-local/install-braincell-lite.ps1 | iex

Option B — download and run locally:

powershell 
.\install-braincell-lite.ps1
Available options:
-TargetPath <path> Install into a different repo (default: current directory)
-SkipCommit Skip the git commit prompt at the end
-Force Overwrite existing .braincell files without asking
2

Answer the commit prompt — you are done

The installer creates all required files and offers to commit them. After committing, open a new Copilot Chat — memory is active immediately.

After installation your repo will contain:

your-repo/
  ├── .braincell/
  │   ├── SKILL.md                 ← full memory skill
  │   ├── braincell-lite-index.json ← fast-search index
  │   └── braincell-lite-notes.json ← initial memory store
  └── .github/
      ├── copilot-instructions.md      ← activation hook
      ├── hooks/
      │   └── braincell-hooks.json      ← SessionStart config
      └── scripts/
          └── braincell-session-start.ps1 ← loads memory at startup

Memory topics

Each topic is a separate JSON file. New topic files are created automatically when needed. Custom topics are allowed.

braincell-lite-notes.json Notes General context, session summaries, diagrams
braincell-lite-decisions.json Decisions Architecture and approach decisions made during sessions
braincell-lite-tasks.json Tasks Tasks tracked with open / done / blocked status
braincell-lite-errors.json Errors Bugs found and fixed — solution pattern captured
braincell-lite-patterns.json Patterns Working commands, reusable code solutions
braincell-lite-facts.json Facts Tech stack, versions, key URLs, project metadata
braincell-lite-preferences.json Preferences Style rules, team conventions, constraints
braincell-lite-[custom].json Custom Any domain you need — e.g. auth, api, infra, security

Privacy

All memories are stored locally in your repository under .braincell/. Nothing is sent to any external service. BrainCell Lite has no network component, no telemetry, and no runtime dependencies.

Memory files are committed to Git with your project — they are part of your codebase, version-controlled and visible to anyone with repo access.

Do not store secrets, passwords, or API tokens in memory files.
